### 目的

>解决来自耦合性、内聚性、可维护性、重用性和灵活性等多方面的问题

### 注意事项

>不要拘泥于某种模式的套路，许多框架在设计时并不会严格按照某种套路来写，重要的是体会设计模式的思路。
